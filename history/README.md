# Project History

This directory contains historical records of all interactions, decisions, and development activities for the AI/Spec-Driven Interactive Book with Integrated RAG Chatbot project.

## Directory Structure

- `prompts/` - Prompt History Records (PHRs) organized by context
  - `constitution/` - Records related to project constitution and governance
  - `module1-ros2/` - Records for ROS 2 Educational Module
  - `module2-digital-twin/` - Records for Digital Twin Simulation Module
  - `module3-ai-brain/` - Records for AI-Robot Brain Module
  - `module4-vla/` - Records for Vision-Language-Action Module
  - `general/` - General project records not specific to a module

## Purpose

The history directory serves as an audit trail and knowledge repository for:
- Design decisions and their rationale
- Implementation plans and changes
- Problem-solving approaches
- Requirements evolution
- Architectural decisions
- Code review records

## Naming Convention

Prompt History Records follow the format:
```
[ID]-[slug].[stage].prompt.md
```

Where:
- `ID` - Sequential identifier
- `slug` - 3-7 word description
- `stage` - One of: constitution, spec, plan, tasks, red, green, refactor, explainer, misc, general

## Usage

This history enables:
- Knowledge transfer between team members
- Understanding of decision context
- Audit trail for compliance
- Learning from past approaches
- Continuity in development efforts
